Image SEO

Images affect both search visibility (Google Images, SERP image results) and Core Web Vitals performance (LCP, CLS). Correct markup, modern formats, and responsive delivery are all required for optimal image SEO.

Complete image markup

Use the <picture> element to serve modern formats (AVIF, WebP) with a JPEG fallback. Include explicit width and height attributes on the <img> to prevent layout shift. For the LCP (hero) image, set fetchpriority="high" to instruct the browser to prioritise the fetch.

Hero image — LCP candidate
HTML 
<picture>
  <!-- AVIF: best compression, modern browsers -->
  <source
    type="image/avif"
    srcset="
      /assets/images/hero-480w.avif  480w,
      /assets/images/hero-800w.avif  800w,
      /assets/images/hero-1200w.avif 1200w
    "
    sizes="(max-width: 600px) 480px, (max-width: 900px) 800px, 1200px"
  />

  <!-- WebP: good compression, broad support -->
  <source
    type="image/webp"
    srcset="
      /assets/images/hero-480w.webp  480w,
      /assets/images/hero-800w.webp  800w,
      /assets/images/hero-1200w.webp 1200w
    "
    sizes="(max-width: 600px) 480px, (max-width: 900px) 800px, 1200px"
  />

  <!-- JPEG: universal fallback -->
  <img
    src="/assets/images/hero-1200w.jpg"
    srcset="
      /assets/images/hero-480w.jpg  480w,
      /assets/images/hero-800w.jpg  800w,
      /assets/images/hero-1200w.jpg 1200w
    "
    sizes="(max-width: 600px) 480px, (max-width: 900px) 800px, 1200px"
    alt="Technical SEO reference documentation interface showing structured data examples"
    width="1200"
    height="630"
    fetchpriority="high"
    loading="eager"
    decoding="sync"
  />
</picture>

Lazy-loading below-fold images

Images that appear below the fold should use loading="lazy" to defer their fetch until the user scrolls near them. Do not apply lazy loading to the LCP image or any image visible in the initial viewport — doing so directly harms Core Web Vitals.

Below-fold image — lazy loaded
HTML 
<picture>
  <source
    type="image/avif"
    srcset="/assets/images/diagram-800w.avif 800w, /assets/images/diagram-400w.avif 400w"
    sizes="(max-width: 600px) 400px, 800px"
  />
  <source
    type="image/webp"
    srcset="/assets/images/diagram-800w.webp 800w, /assets/images/diagram-400w.webp 400w"
    sizes="(max-width: 600px) 400px, 800px"
  />
  <img
    src="/assets/images/diagram-800w.jpg"
    alt="Diagram showing the relationship between canonical tags and duplicate URLs"
    width="800"
    height="450"
    loading="lazy"
    decoding="async"
  />
</picture>

Alt text guidelines

Alt text is read by screen readers, displayed when images fail to load, and used by Google to understand image content. Write alt text that describes what is visually meaningful about the image in the context of the surrounding content.

Image type Alt text approach Example
Informational / editorial Describe the content and its relevance to the surrounding text alt="Bar chart comparing LCP scores before and after preload optimisation"
Product image Include product name, key attributes, and variant details alt="Blue leather notebook A5 with elastic closure, front view"
Logo State the brand name; do not describe the graphic alt="SEO Reference"
Decorative / background Empty string — do not describe decorative content alt=""
Icon with adjacent text label Empty string — the label provides the accessible name alt=""
Image link (no text) Describe the link destination, not the image appearance alt="Go to homepage"

Filename conventions

Google reads filenames as a relevance signal for Google Images ranking. Use descriptive, hyphen-separated slugs that reflect the image content. Avoid generic names like image1.jpg or CMS-generated hashes.

Avoid
  • IMG_4821.jpg
  • image1.png
  • photo.webp
  • DSC00123_final_v2.jpg
  • untitled-export.avif
  • screenshot 2025-03.png (spaces)
Use
  • technical-seo-reference-hero.avif
  • canonical-tag-diagram.webp
  • organization-schema-knowledge-panel.jpg
  • lcp-before-after-chart.png
  • breadcrumb-serp-example.webp
  • robots-txt-crawl-budget.avif
