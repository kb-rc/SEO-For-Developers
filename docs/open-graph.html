Open Graph & Social Sharing

Open Graph (OG) meta tags control how a URL renders when shared on Facebook, LinkedIn, Slack, iMessage, and other platforms. Twitter/X uses its own twitter:card protocol but falls back to OG when Twitter-specific tags are absent.

Required OG tags

The five required Open Graph properties are og:type, og:url, og:title, og:description, and og:image. The following is the complete OG implementation from index.html:

index.html - Open Graph
HTML 
<!-- Open Graph -->
<meta property="og:type"        content="website" />
<meta property="og:url"         content="https://kb-rc.github.io/SEO-For-Developers/" />
<meta property="og:title"       content="Technical SEO Reference for Developers" />
<meta property="og:description" content="A complete technical SEO reference for developers. Covers title tags, structured data, sitemaps, robots.txt, and more." />
<meta property="og:image"       content="https://kb-rc.github.io/SEO-For-Developers/assets/images/og-cover.png" />
<meta property="og:image:width"  content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:site_name"   content="SEO For Developers" />

Twitter / X Card

Twitter/X reads its own twitter:* meta tags first, and falls back to OG equivalents when they are missing. Always include both sets for maximum compatibility. The summary_large_image card type renders a full-width image above the title and description.

index.html - Twitter Card
HTML 
<!-- Twitter Card -->
<meta name="twitter:card"        content="summary_large_image" />
<meta name="twitter:site"        content="@seodocsdev" />
<meta name="twitter:title"       content="Technical SEO Reference for Developers" />
<meta name="twitter:description" content="A complete technical SEO reference for developers." />
<meta name="twitter:image"       content="https://kb-rc.github.io/SEO-For-Developers/assets/images/og-cover.png" />

Article OG type

For blog posts, news articles, and documentation pages, set og:type to article and include article-specific metadata. This allows Facebook and LinkedIn to categorize the content correctly and may influence how feeds surface the link.

docs/[article].html - Article OG type
HTML 
<meta property="og:type"                  content="article" />
<meta property="og:url"                   content="https://kb-rc.github.io/SEO-For-Developers/docs/title-tags.html" />
<meta property="og:title"                 content="Title Tags - SEO Reference" />
<meta property="og:description"           content="How to write effective HTML title tags for SEO." />
<meta property="og:image"                 content="https://kb-rc.github.io/SEO-For-Developers/assets/images/og-cover.png" />
<meta property="article:published_time"   content="2026-03-26T00:00:00Z" />
<meta property="article:modified_time"    content="2026-03-26T00:00:00Z" />
<meta property="article:author"           content="https://kb-rc.github.io/SEO-For-Developers/" />
<meta property="article:section"          content="On-Page Tags" />
<meta property="article:tag"              content="Title Tags" />
<meta property="article:tag"              content="Technical SEO" />

OG image specifications

The OG image is the most visible element of any shared link. Incorrect dimensions result in cropping, letterboxing, or the platform refusing to display the image entirely.

Property Recommended value Notes
Dimensions 1200 × 630 px Minimum 600 × 315 px. 1200 × 630 renders sharply on retina displays.
Aspect ratio 1.91:1 Facebook, LinkedIn, and X all expect this ratio for summary_large_image.
Format PNG or JPEG PNG for graphics and text; JPEG for photographs. WebP is not universally supported by OG scrapers.
File size Under 300 KB Facebook caches OG images aggressively. Large files cause slow preview rendering and may be skipped.
Safe zone 100 px from each edge Keep all text and logos within the safe zone; cropping varies by platform and viewport.
URL Absolute URL Never use relative paths. Social crawlers fetch images independently without a page context.
Use absolute URLs for all OG image and URL values. Social platform crawlers fetch resources independently and do not resolve relative URLs against a base URL. A relative path such as /assets/images/og-cover.png will fail silently - the platform will display no image in the preview.
