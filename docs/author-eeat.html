Author / E-E-A-T Schema

E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness) is Google's quality evaluation framework. Person schema provides the structured signal that associates authored content with a verifiable entity, supporting all four dimensions.

Author page (Person schema)

A dedicated author profile page is the foundation of schema-based E-E-A-T signaling. The page should include the author's credentials, areas of expertise, and links to authoritative external profiles. The Person schema on this page then provides a stable @id URL that can be referenced across all articles the author has written.

Without a dedicated author page, the author property in Article schema can only contain inline text, which Google cannot verify against any entity record. A resolvable @id URL transforms an anonymous byline into a Knowledge Graph entity.

Person schema (used in <head> of all doc pages)
JSON-LD 
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Person",
  "@id": "https://kb-rc.github.io/SEO-For-Developers/#author",
  "name": "SEO Reference",
  "url": "https://kb-rc.github.io/SEO-For-Developers/",
  "jobTitle": "Technical SEO Documentation",
  "knowsAbout": [
    "Technical SEO",
    "Structured Data",
    "Core Web Vitals",
    "JavaScript SEO",
    "Crawling and Indexing"
  ]
}
</script>

Person schema properties for E-E-A-T

Property E-E-A-T Dimension Notes
@id All Persistent entity URL. Use the author profile page URL with a fragment: /about/author-name#person.
name Authority Full name as it appears in bylines and external sources.
jobTitle Expertise Professional title or role. Establishes subject-matter expertise.
knowsAbout Expertise Array of topics the author is an expert in. Aligns content topical authority with author knowledge.
sameAs Authority, Trust Links to LinkedIn, GitHub, Google Scholar, and other verifiable profiles.
hasCredential Expertise, Trust EducationalOccupationalCredential entities listing degrees, certifications, and professional qualifications.
worksFor Authority References the Organization entity via @id, connecting author authority to brand authority.

Referencing author in Article schema

Every article or documentation page should include an Article (or TechArticle) schema that references the author entity via @id. Using the @id reference — rather than repeating all author properties inline — keeps the markup concise and allows Google to resolve the full entity definition from the author profile page.

TechArticle schema with author @id reference
JSON-LD 
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "TechArticle",
  "headline": "Author Schema & E-E-A-T Signals",
  "description": "How to implement Person schema for E-E-A-T signals.",
  "url": "https://kb-rc.github.io/SEO-For-Developers/docs/author-eeat.html",
  "datePublished": "2025-01-01",
  "dateModified": "2026-03-26",
  "author": {
    "@id": "https://kb-rc.github.io/SEO-For-Developers/#author"
  },
  "publisher": {
    "@id": "https://kb-rc.github.io/SEO-For-Developers/#organization"
  }
}
</script>

Reviewer markup for YMYL content

For Your Money or Your Life (YMYL) content — health, finance, legal topics — Google places extra scrutiny on E-E-A-T signals. Adding a reviewedBy property with a qualified reviewer's Person schema strengthens trustworthiness signals on these pages.

YMYL article with reviewer
JSON-LD 
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Understanding Tax Deductions for Freelancers",
  "author": {
    "@type": "Person",
    "name": "Jane Smith",
    "url": "https://example.com/authors/jane-smith"
  },
  "reviewedBy": {
    "@type": "Person",
    "name": "Dr. Alan Chen, CPA",
    "jobTitle": "Certified Public Accountant",
    "url": "https://example.com/reviewers/alan-chen"
  }
}
