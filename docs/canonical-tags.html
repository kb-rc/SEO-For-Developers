Canonical Tags

<link rel="canonical"> tells search engines which URL is the authoritative version when duplicate or near-duplicate content exists across multiple URLs. Link equity and ranking signals are consolidated to the canonical URL.

Self-referencing canonical

Every page — including pages with no known duplicates — should carry a self-referencing canonical tag. Without it, search engines may independently decide which version of the URL to treat as canonical when parameters, trailing slashes, or protocol variants are present (e.g. https vs http). The canonical used in index.html:

index.html
HTML 
<link rel="canonical" href="https://kb-rc.github.io/SEO-For-Developers/" />

Duplicate content consolidation

When the same or substantially similar content appears at multiple URLs — for example, a paginated page with URL parameters — the duplicate variants should point to the primary URL via a canonical tag. This consolidates ranking signals without removing the alternative URLs from the site.

docs/title-tags.html?ref=sidebar (duplicate variant)
HTML 
<!-- On the parameterized duplicate, point back to the canonical URL -->
<link rel="canonical"
      href="https://kb-rc.github.io/SEO-For-Developers/docs/title-tags.html" />

<!-- For paginated content, page 2 points to itself (not page 1) -->
<link rel="canonical"
      href="https://kb-rc.github.io/SEO-For-Developers/docs/title-tags.html?page=2" />

Cross-domain canonical

When the same content is syndicated or published on a second domain, the syndicated version should point its canonical tag back to the original source. This tells Google which site owns the content and prevents the syndication from competing with or outranking the original.

syndicated-site.com/title-tags (cross-domain duplicate)
HTML 
<!--
  Published on: https://syndicated-site.com/title-tags
  Original at:  https://kb-rc.github.io/SEO-For-Developers/docs/title-tags.html
-->
<link rel="canonical"
      href="https://kb-rc.github.io/SEO-For-Developers/docs/title-tags.html" />
Canonical is a hint, not a directive. Google may override the canonical tag if it detects conflicting signals — for example, if the canonical URL returns a non-200 status code, is blocked by robots.txt, or has a noindex directive. Always ensure the canonical URL is accessible, indexable, and returns HTTP 200. Do not point canonicals at redirect chains.
