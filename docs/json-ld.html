JSON-LD Structured Data

JSON-LD (JavaScript Object Notation for Linked Data) is a method of encoding Schema.org structured data as a <script type="application/ld+json"> block. Search engines use it to understand page content and generate rich results — enhanced SERP features such as FAQ accordions, star ratings, breadcrumbs, and knowledge panels.

Placement

Google recommends placing JSON-LD blocks in the <head> element for optimal performance. However, Google also processes JSON-LD found anywhere in the <body>. Multiple <script type="application/ld+json"> blocks on a single page are fully valid — each represents a separate schema entity. All pages on this site include three JSON-LD blocks in <head>: Organization, Person/E-E-A-T, and BreadcrumbList, with page-specific schemas such as FAQPage added as a fourth block.

Article schema

The TechArticle type is appropriate for technical documentation pages. It extends Article with fields relevant to technical content. The following example uses the kb-rc.github.io domain and matches the structure used across this site's documentation pages.

docs/json-ld.html — TechArticle JSON-LD
JSON-LD 
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "TechArticle",
  "headline": "JSON-LD Structured Data Basics",
  "description": "How to implement JSON-LD structured data for rich results.",
  "url": "https://kb-rc.github.io/SEO-For-Developers/docs/json-ld.html",
  "datePublished": "2026-03-26",
  "dateModified": "2026-03-26",
  "author": {
    "@type": "Person",
    "@id": "https://kb-rc.github.io/SEO-For-Developers/#author"
  },
  "publisher": {
    "@type": "Organization",
    "@id": "https://kb-rc.github.io/SEO-For-Developers/#organization"
  },
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "https://kb-rc.github.io/SEO-For-Developers/docs/json-ld.html"
  },
  "proficiencyLevel": "Beginner",
  "keywords": [
    "JSON-LD",
    "Structured Data",
    "Schema.org",
    "Rich Results",
    "Technical SEO"
  ]
}
</script>

Dynamic JSON-LD injection

For single-page applications or client-side navigation, JSON-LD blocks must be updated as the page content changes. The injectJSONLD utility in assets/js/seo.js handles this by creating a new <script> tag and removing any previous block with the same ID to prevent duplicate schema entries accumulating in <head>.

The following is the exact function as implemented in assets/js/seo.js:

assets/js/seo.js — injectJSONLD
JS 
/**
 * Injects a JSON-LD block into <head>.
 * Removes any previous block with the same id to avoid duplicates.
 * @param {Object} schema - Schema.org object
 * @param {string} id     - Unique ID for the script tag
 */
function injectJSONLD(schema, id) {
  if (id === undefined) id = 'jsonld-main';
  const existing = document.getElementById(id);
  if (existing) existing.remove();

  const script = document.createElement('script');
  script.type = 'application/ld+json';
  script.id   = id;
  script.text = JSON.stringify(schema, null, 2);
  document.head.appendChild(script);
}

// Usage:
injectJSONLD({
  "@context": "https://schema.org",
  "@type": "TechArticle",
  "headline": "JSON-LD Structured Data Basics"
  // ... full schema object
});
