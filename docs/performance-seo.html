Performance SEO

Core Web Vitals (LCP, CLS, INP) are direct ranking signals since 2021. Good thresholds: LCP under 2.5s, CLS under 0.1, INP under 200ms. Performance optimisation is therefore both a user experience requirement and an SEO requirement.

Resource preloading

Preconnect and preload hints reduce the critical-path latency for resources the browser will need. Preconnect establishes early TCP/TLS connections to external origins. Preload fetches high-priority resources - fonts, critical CSS, LCP images - before the parser would normally discover them.

The following is the resource hint pattern used in the <head> of every page on this site:

index.html - resource hints
HTML 
<!-- 1. Preconnect to font origins as early as possible -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />

<!-- 2. Load font stylesheet (render-blocking, but preconnect reduces latency) -->
<link
  href="https://fonts.googleapis.com/css2?family=IBM+Plex+Mono:wght@400;500&family=Syne:wght@400;500;600;700&family=DM+Sans:ital,opsz,wght@0,9..40,300;0,9..40,400;0,9..40,500;1,9..40,300&display=swap"
  rel="stylesheet"
/>

<!-- 3. Load site stylesheet -->
<link rel="stylesheet" href="assets/css/style.css" />

<!-- 4. Preload LCP image (if known at build time) -->
<link
  rel="preload"
  as="image"
  href="assets/images/hero-1200w.avif"
  imagesrcset="assets/images/hero-480w.avif 480w, assets/images/hero-800w.avif 800w, assets/images/hero-1200w.avif 1200w"
  imagesizes="(max-width: 600px) 480px, (max-width: 900px) 800px, 1200px"
  fetchpriority="high"
/>

Script loading strategies

JavaScript blocks HTML parsing when loaded synchronously. Use defer, async, or type="module" to prevent render-blocking scripts. The assets/js/seo.js file on this site uses defer, which executes after HTML parsing completes but before DOMContentLoaded, preserving execution order.

Script loading comparison
HTML 
<!-- Render-blocking: avoid for non-critical scripts -->
<script src="script.js"></script>

<!-- defer: parses in parallel, executes after HTML parsed, in order -->
<!-- Best for scripts that depend on the DOM and need to run in order -->
<script src="assets/js/seo.js" defer></script>

<!-- async: fetches in parallel, executes immediately when ready -->
<!-- Best for independent scripts like analytics that do not need order -->
<script src="analytics.js" async></script>

<!-- type="module": deferred by default, strict mode, supports import/export -->
<script type="module" src="app.js"></script>

<!-- Inline critical script: use sparingly, only for truly critical logic -->
<script>
  // Theme initialisation must run before first paint to prevent flash
  const theme = localStorage.getItem('theme') || 'dark';
  document.documentElement.setAttribute('data-theme', theme);
</script>

Preventing CLS

Cumulative Layout Shift measures unexpected visual instability during page load. A CLS score above 0.1 is classified as needing improvement. The most common fixable causes are images without explicit dimensions and web fonts causing a flash of unstyled text.

Cause Fix
Images without width and height attributes Always declare width and height on <img>. The browser reserves the correct space before the image loads, preventing a reflow.
Web fonts causing FOUT (flash of unstyled text) Use font-display: swap in @font-face declarations. This renders text in a fallback font immediately and swaps to the web font once loaded, avoiding invisible-text flicker.
Dynamically injected banners or alerts above content Reserve space with a fixed-height container or use CSS min-height on the injection target before the content loads.
Ad and embed containers without dimensions Set explicit width and height (or aspect-ratio) on the container element before the third-party script injects the ad or iframe.
Late-loading custom fonts shifting text metrics Use size-adjust and ascent-override in fallback font declarations to match the metrics of the web font, minimising the layout shift on swap.
CSS - image dimensions and font-display
CSS 
/* Prevent CLS from images: always reserve aspect ratio space */
img {
  max-width: 100%;
  height: auto;        /* preserves aspect ratio when width is constrained */
  display: block;      /* removes inline baseline gap */
}

/* font-display: swap prevents invisible text during font load */
@font-face {
  font-family: 'Syne';
  font-style: normal;
  font-weight: 400 700;
  font-display: swap;  /* render fallback immediately, swap when ready */
  src: url('/fonts/syne.woff2') format('woff2');
}

/* Reserve space for ad/embed containers before script injection */
.ad-container {
  min-height: 250px;
  width: 100%;
}

Optimising INP (Interaction to Next Paint)

INP replaced FID (First Input Delay) as a Core Web Vital in March 2024. It measures the full duration of all interactions - click, keypress, tap - from the moment of interaction to the next visual update. The good threshold is under 200ms.

Key techniques: break up long tasks with scheduler.yield() or setTimeout(0), avoid synchronous layout operations in event handlers, and reduce JavaScript bundle size to minimise main-thread blocking time after load.
