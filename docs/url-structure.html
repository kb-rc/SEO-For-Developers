URL Structure Best Practices

URLs are visible ranking signals and usability touchpoints. A clean, keyword-rich URL reinforces topical relevance, passes crawl budget efficiently, and increases click-through rates in SERPs by giving users immediate context about the destination page.

Rules

Rule Correct Avoid
Use hyphens as word separators /docs/canonical-tags.html /docs/canonical_tags.html, /docs/canonicaltags.html
Lowercase only /docs/title-tags.html /docs/Title-Tags.html, /DOCS/TITLE-TAGS.HTML
Include the primary keyword /docs/breadcrumb-schema.html /docs/page-12.html, /docs/p?id=987
No dates in evergreen content URLs /docs/canonical-tags.html /docs/2024/03/canonical-tags.html
Consistent trailing slash convention /docs/robots-txt.html (always without trailing slash on .html files) Mixing /docs/robots-txt.html and /docs/robots-txt.html/
Shallow hierarchy (max 3 path segments for primary content) /docs/json-ld.html /docs/structured-data/schemas/json-ld/basics.html
No special characters or encoded spaces /docs/open-graph.html /docs/open%20graph.html, /docs/open_graph&v=2.html

Canonical redirect handling

When a URL changes - due to restructuring, slug updates, or content consolidation - a 301 redirect from the old URL to the new URL preserves the ranking signals accumulated at the old URL. Server-side 301s pass PageRank; client-side JavaScript redirects do not.

The assets/js/seo.js file on this site contains a REDIRECTS object for client-side fallback routing. This handles legacy URLs that were used before the current URL structure was finalised, specifically for cases where GitHub Pages cannot serve server-side redirects.

assets/js/seo.js - REDIRECTS object
JS 
/**
 * Client-side redirect fallback.
 * Server-side 301 redirects are always preferred.
 * Use this only for client-side SPA routing fallbacks.
 * Configuring 301s in nginx/Caddy/Vercel/GitHub Pages is the correct approach.
 */
var REDIRECTS = {
  '/SEO-For-Developers/blog/json-ld-guide.html': '/SEO-For-Developers/docs/json-ld.html',
  '/SEO-For-Developers/docs/structured_data':    '/SEO-For-Developers/docs/json-ld.html'
};

var redirect = REDIRECTS[window.location.pathname];
if (redirect) window.location.replace(redirect);
For server-side redirects, configure 301s in your server config or netlify.toml. Client-side window.location.replace() does not pass PageRank and introduces a JavaScript dependency in the redirect chain.
