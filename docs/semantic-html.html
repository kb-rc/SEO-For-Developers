Semantic HTML for SEO

Semantic HTML uses elements that carry intrinsic meaning - <article>, <nav>, <main>, <header>, <footer> - rather than generic <div> and <span> wrappers. Correct landmark usage helps Googlebot understand page structure and improves accessibility for screen reader users.

Document structure

The following shows the full landmark structure used on every documentation page on this site. Each landmark element maps to a WAI-ARIA role, which both assistive technologies and search engines use to understand the page's information architecture.

docs/[page].html - body structure
HTML 
<body>

  <!-- Site-wide header: logo, primary nav, search, theme toggle -->
  <header class="header" role="banner">
    <nav class="header-nav" role="navigation" aria-label="Primary">
      <!-- search widget -->
    </nav>
  </header>

  <!-- Three-column layout: sidebar + main + TOC -->
  <div class="layout">

    <!-- Left sidebar: documentation navigation -->
    <nav class="sidebar" role="navigation" aria-label="Documentation">
      <!-- nav groups and links -->
    </nav>

    <!-- Primary content region -->
    <main class="main" role="main" id="main-content">
      <div class="content">

        <!-- Self-contained documentation article -->
        <article aria-labelledby="page-h1">
          <h1 id="page-h1">Page Title</h1>

          <!-- Thematic section: groups related content within the article -->
          <section aria-labelledby="section-heading">
            <h2 id="section-heading">Section Title</h2>
            <!-- section content -->
          </section>

        </article>

      </div>
    </main>

    <!-- Right rail: on-this-page table of contents -->
    <aside class="toc" aria-label="On this page">
      <!-- TOC generated by seo.js -->
    </aside>

  </div>

</body>

Heading hierarchy

Headings create the document outline that search engines use to understand content structure. Follow these rules on every page:

  • One h1 per page. The h1 topic must align with the <title> tag. It is the primary signal for the page's core topic.
  • h2 for major sections. Each h2 introduces a top-level topic that supports the h1. All h2s on this page (Document structure, Heading hierarchy, Landmark elements, Microdata) are direct subtopics of "Semantic HTML for SEO".
  • h3 for subsections. Use h3 inside an h2 section for further subdivision. Never skip levels - jumping from h2 to h4 breaks the document outline.
  • Never use headings for visual styling. If you need large bold text that is not a structural heading, use CSS on a <p> or <span>. Using h2 for decorative pull-quotes corrupts the document outline.

Landmark elements

Element ARIA Role Usage
<header> banner (when top-level) Site-wide header containing logo and primary navigation. Only one banner landmark per page.
<nav> navigation Navigation landmark. Use aria-label to distinguish multiple nav elements (e.g., "Primary", "Documentation", "Breadcrumb").
<main> main Primary content of the page. Only one per page. Screen readers provide a skip link to this landmark.
<aside> complementary Content tangentially related to the main content (table of contents, related links, ads). Use aria-label to describe its purpose.
<footer> contentinfo (when top-level) Site-wide footer with copyright, legal links, and secondary navigation.
<article> article Self-contained, independently distributable content. Use for blog posts, documentation pages, news articles, and comments.
<section> region (when labelled) Thematic grouping within a page that would not make sense in isolation. Always provide a heading or aria-label.

Microdata inline markup

While JSON-LD is preferred for most structured data, microdata can reinforce schema markup inline when the structured content is already present in the HTML. Microdata attributes - itemscope, itemtype, itemprop - annotate existing visible elements without duplicating data in a separate script block.

Article with microdata
HTML 
<article
  itemscope
  itemtype="https://schema.org/TechArticle"
>
  <h1 itemprop="headline">Semantic HTML for SEO</h1>

  <p>
    By
    <span
      itemprop="author"
      itemscope
      itemtype="https://schema.org/Person"
    >
      <a
        itemprop="url"
        href="https://kb-rc.github.io/SEO-For-Developers/"
      >
        <span itemprop="name">SEO Reference</span>
      </a>
    </span>
  </p>

  <meta itemprop="datePublished" content="2025-01-01" />
  <meta itemprop="dateModified" content="2026-03-26" />

  <div itemprop="articleBody">
    <!-- Article content -->
  </div>
</article>
