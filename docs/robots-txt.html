robots.txt

robots.txt is a plain-text file at the root of your domain that instructs compliant crawlers which paths they may and may not access. It is part of the Robots Exclusion Protocol (REP), a standard supported by all major search engines.

File location

The file must be accessible at the root of the domain — not a subdirectory. For this site, it lives at:

https://kb-rc.github.io/SEO-For-Developers/robots.txt

Each subdomain requires its own robots.txt. A file placed at /blog/robots.txt or any subdirectory path is not recognized by crawlers.

Syntax

The following is the exact robots.txt file deployed at the root of this site. It allows all standard crawlers, excludes static assets from crawling, and blocks known AI training crawlers.

robots.txt
TXT 
# robots.txt — https://kb-rc.github.io/SEO-For-Developers/
# Allow all crawlers to access the full site
User-agent: *
Allow: /

# Block crawlers from static assets (no SEO value)
Disallow: /assets/

# Block AI training crawlers
User-agent: GPTBot
Disallow: /

User-agent: CCBot
Disallow: /

User-agent: anthropic-ai
Disallow: /

User-agent: Google-Extended
Disallow: /

# Sitemap reference
Sitemap: https://kb-rc.github.io/SEO-For-Developers/sitemap.xml

Key rules

Directive Behaviour
User-agent: * Applies the following rules to all crawlers that do not have a more specific User-agent block.
Allow: / Permits crawling of the entire site. An empty or missing Allow is equivalent; this line is explicit for clarity.
Disallow: /assets/ Blocks crawling of all paths under /assets/. CSS, JS, and image files provide no direct indexing value and can waste crawl budget on large sites.
User-agent: GPTBot Targets OpenAI's training crawler specifically. More restrictive than the wildcard block above.
Disallow: / When placed under a named User-agent, blocks that specific crawler from all paths on the site.
Sitemap: Provides the absolute URL of the XML sitemap. Crawlers use this to discover all URLs without relying solely on crawling links. Multiple Sitemap: lines are valid.
robots.txt does not prevent indexing. Blocking a URL via Disallow only prevents crawling — it does not stop the URL from appearing in search results if external sites link to it. To exclude a page from the index, use <meta name="robots" content="noindex"> or an X-Robots-Tag: noindex response header, and ensure the page is still crawlable so Googlebot can read the directive.
