Organization Schema

Organization schema tells Google about your brand entity - name, logo, contact details, and social profiles. It contributes to Knowledge Panel eligibility and brand disambiguation in the Knowledge Graph.

Organization JSON-LD

The following is the exact Organization schema used on this site. It is placed in the <head> of every page to ensure consistent entity signaling across all URLs.

docs/organization-schema.html
JSON-LD 
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "@id": "https://kb-rc.github.io/SEO-For-Developers/#organization",
  "name": "SEO For Developers",
  "url": "https://kb-rc.github.io/SEO-For-Developers/",
  "description": "Technical SEO reference documentation for developers.",
  "logo": {
    "@type": "ImageObject",
    "url": "https://kb-rc.github.io/SEO-For-Developers/assets/images/logo.png",
    "width": 512,
    "height": 512
  },
  "sameAs": [
    "https://github.com/kb-rc"
  ]
}
</script>
The @id property creates a persistent entity identifier in the Knowledge Graph. Use a consistent @id across all pages to link your Organization entity to Article schemas via the publisher property.

Key properties

Property Type Purpose
@id URL Persistent entity identifier. Use the pattern https://example.com/#organization. Must remain stable - never change it once established.
name Text Official brand name. Match exactly how it appears in other authoritative sources.
url URL The canonical homepage URL. Must match your canonical tag and sitemap.
logo ImageObject Google uses this image in Knowledge Panels and search results. Recommended: square image, minimum 112x112px, maximum 1MB.
description Text A brief description of the organization. Surfaces in Knowledge Panels when no Wikipedia excerpt is available.
sameAs URL[] Social profile URLs and authoritative directory listings. Helps Google disambiguate your entity from other organizations with similar names.

sameAs social profiles

The sameAs array should include all authoritative profiles where your organization is officially present. These act as entity disambiguation signals in the Knowledge Graph.

Organization sameAs example
JSON 
"sameAs": [
  "https://github.com/your-org",
  "https://www.linkedin.com/company/your-org",
  "https://twitter.com/your-org",
  "https://en.wikipedia.org/wiki/Your_Organization"
]

Linking Organization to Article schemas

Once your Organization @id is established, reference it in Article schemas via the publisher property. This creates a verifiable relationship between your content and your brand entity.

Article schema with publisher reference
JSON-LD 
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Organization Schema - Brand Entity Markup",
  "publisher": {
    "@id": "https://kb-rc.github.io/SEO-For-Developers/#organization"
  },
  "author": {
    "@id": "https://kb-rc.github.io/SEO-For-Developers/#author"
  }
}
