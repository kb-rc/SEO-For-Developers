FAQ Schema

The FAQPage schema type marks up a page that contains a list of questions and their answers. When Google recognizes the schema and the page meets eligibility criteria, it may display an expandable FAQ accordion directly in the SERP - occupying additional vertical space and increasing visibility without requiring a click to the page.

Eligibility requirements

  • Visible content only. Every question and answer marked up in the JSON-LD must be visibly rendered in the page HTML. Google will not grant a rich result for content that is hidden, collapsed without interaction, or rendered only via JavaScript after the page loads.
  • One accepted answer per question. FAQPage schema is intended for pages where each question has a single definitive answer. Pages with community voting or multiple competing answers should use QAPage instead.
  • Informational intent. Google reserves FAQPage rich results for genuinely helpful, informational content. Pages with strong commercial intent - product pages, lead generation forms, or FAQ sections used primarily to promote a service - are unlikely to receive the rich result treatment and may receive a manual action if the schema is considered deceptive.

FAQPage JSON-LD

The following is the exact FAQPage JSON-LD block implemented in index.html of this site. It marks up three questions that are rendered as visible content on the index page.

index.html - FAQPage JSON-LD
JSON-LD 
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is a canonical tag used for?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "A canonical tag tells search engines which URL is the authoritative version of a page when duplicate or near-duplicate content exists across multiple URLs."
      }
    },
    {
      "@type": "Question",
      "name": "What is the difference between robots.txt and noindex?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "robots.txt blocks crawlers from accessing a URL entirely. noindex allows crawling but instructs the crawler not to include the page in search results. Use robots.txt to save crawl budget; use noindex to exclude pages you still need crawled."
      }
    },
    {
      "@type": "Question",
      "name": "Does page speed affect SEO rankings?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Yes. Google uses Core Web Vitals (LCP, CLS, INP) as ranking signals. Preloading critical assets, lazy-loading below-fold images, and serving next-gen image formats directly improve these metrics."
      }
    }
  ]
}
</script>
The FAQPage JSON-LD shown above is the exact schema implemented in index.html of this site. The three questions - canonical tags, robots.txt vs noindex, and page speed - are also rendered as visible HTML content on the index page, satisfying Google's requirement that all marked-up Q&A content must be present in the rendered HTML.
